# 🏗️ 技術選定理由 (Architecture)

本プロダクト（診療材料新規採用申請アプリ）で採用した技術スタックと、その選定理由・背景について記述します。

---

## 1. フロントエンド / バックエンド
* **Next.js (App Router)**
  * **選定理由:** 画面表示の高速化（SSR/ISR）および API Routes（Route Handlers）によるフロント・バックエンドの一元管理が可能なため。

## 2. データベース / ORM
* **PostgreSQL / Prisma**
  * **選定理由:** 構造化データ（申請・承認フロー）の整合性を担保するためのリレーショナルデーターベースとして採用。

## 3. テスト / 自動化
* **Playwright**
  * **選定理由:** 複数ブラウザ（Chromium, Firefox, WebKit）に対応した高速かつ安定した E2E テストを構築するため。
* **GitHub Actions**
  * **選定理由:** プルリクエスト作成時やメインブランチへのマージ時に、自動で E2E テストを実行する CI 環境を構築するため。
