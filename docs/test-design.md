# 🧪 テスト設計思想 (Test Design)

JSTQB（ソフトウェアテスト資格）の知識をベースにしたテスト計画および自動テスト（Playwright）の運用ルールについて記述します。

---

## 1. テスト観点と設計技法 (JSTQB)
* **同値分割・境界値分析:** フォーム入力（数量、文字数制限など）における正常系・異常系の洗い出し。
* **状態遷移テスト:** 申請ステータス（申請→承認/却下）のフロー検証。

## 2. E2Eテスト自動化方針 (Playwright)
* **クリティカルパスの優先化:** ユーザーの主要な利用フロー（ログイン → 申請作成 → 承認）を最優先で自動化。
* **保守性の高め方:** Page Object Model (POM) デザインパターンの適用によるメンテナンス性の向上。

## 3. CI/CD パイプライン (GitHub Actions)
* PR（プルリクエスト）トリガーでの自動テスト実行。
* テスト失敗時のスクリーンショット・トレースログの作成・保存方針。
